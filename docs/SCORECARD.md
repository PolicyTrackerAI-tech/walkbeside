# Metrics & the go/no-go scorecard

The living tracker for [OPERATING_PLAN.md](OPERATING_PLAN.md) Part 10. Review the
dashboard weekly; review the go/no-go monthly. Update the "current reading" at
the bottom as you go.

## North-star metric

**Families meaningfully helped per month** — used a tool, got a fair-price
answer, or were served through an institution. It captures the mission and
predicts everything else. One number; track it every month.

## The weekly dashboard

Each week, ask the three questions: **did I grow reach, move an institution
toward paying, and deepen the data?** If a row didn't move, that's the signal.

| Area | Metric | How to read it |
|---|---|---|
| **Reach** | Organic visitors, tool uses, AI/press citations, email subs | Is the free source-of-truth pulling people in for free? |
| **Service** | At-need cases run, avg savings/family, satisfaction score | Is L2 producing proof + the outcomes data? |
| **Data** | Homes covered, GPLs collected, outcomes records captured | Is the moat deepening? (Outcomes records are the gold.) |
| **Sales** | Hospice contacts, calls booked, pilots live, paid contracts (MRR) | Is an institution moving toward paying? (The survival metric.) |
| **Authority** | Index releases, press pickups, backlinks, Wikipedia citations | Are we becoming the cited standard? |

Weekly targets are set by you each Monday (e.g., 15 new hospice contacts, 5
calls booked, 2 guides shipped). Put ≥1 institutional move in every day's top 3.

## The go/no-go scorecard (the quit-the-job decision)

Don't quit on feeling or on consumer sales. Go full-time only when these turn
green — from evidence and stability, not hope.

| Signal | Green-light threshold | Status |
|---|---|---|
| **Institutional revenue** | ≥1 paid recurring contract; a 2nd in pilot | 🔴 not started |
| **Repeatable motion** | Converting >1 in ~5 hospice conversations | 🔴 no conversations yet |
| **Family outcomes** | Documented savings + strong satisfaction across ~10+ cases | 🟡 instrumentation built, 0 cases recorded |
| **Free-tool traction** | Organic traffic + first AI/press citations climbing | 🟡 L1 built, analytics not yet instrumented |
| **Runway** | Revenue covers costs **OR** ~12–18 mo savings **OR** a raise | ⚪ founder to assess |

Legend: 🟢 green · 🟡 partial · 🔴 not yet · ⚪ founder input needed.

## Cadence

- **Daily:** pick 3 priorities; ≥1 must move an institution toward paying.
- **Weekly:** personal review against the dashboard; set next week's targets.
- **Monthly:** dashboard + go/no-go vs this scorecard; answer one strategic
  question.

## Instrumentation gaps (what we can't measure yet)

To make this dashboard real, these need wiring (none block the plan, but they
gate honest measurement):

- **Reach analytics** — no product analytics on the free tools yet (visitors,
  tool uses, email subs). Add lightweight analytics (privacy-respecting).
- **Outcomes data** — the outcomes migration must be applied before Service/Data
  rows populate (see [ROADMAP.md](ROADMAP.md) P1).
- **Sales CRM** — even a spreadsheet; track every hospice through the 8 stages
  (see `HOSPICE_GTM.md`).
- **Authority** — manual for now: log Index releases, press, backlinks.
